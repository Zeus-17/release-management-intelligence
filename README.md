## What it does

The tool covers the full release lifecycle — from first planning to formal PIR sign-off:

| Stage | Capability |
|---|---|
| **Planning** | Release type, delivery models, scope register, golden thread, risk register, deployment runbook, release dependencies |
| **Gates** | 8 quality gates auto-generated by release type; entry/exit checklists; sign-off with evidence; delivery-model-specific guidance |
| **Defects** | Enriched triage — type, severity, priority, source, team, component, environment found; custom severity scales; Go/No-Go readiness |
| **Environments** | DEV→SIT→UAT→PRE-PROD→PROD promotion pipeline; health status; config drift warnings |
| **Go/No-Go** | 5-step structured wizard producing a formal, timestamped decision record with audit trail |
| **Hypercare** | Post-deployment countdown timer, monitoring checklist, rollback trigger criteria, All Clear sign-off |
| **Reporting** | Five audience-tailored outputs: Executive, Account Executive, Release Manager, Team, PIR Document |
| **Integration** | 12 tool integrations; live webhook simulation; JS API (`window.RMTool`); URL parameters |
| **Calendar** | Blackout periods: change freeze, financial reporting, maintenance, bank holidays, offshore unavailability |
| **Governance** | Separate Retrospective, Scope Review and PIR tabs; configurable PIR name (AAR, Post Mortem etc.) |

## Key features

- **Release Readiness Score** — 0–100 animated ring per release calculated from gates, defects, environments and sign-offs
- **Deterministic AI Intelligence** — 17 pattern rules fire as you type (payment/regulatory, auth, database, mobile, security, GDPR keywords and more). No external API. No data leaves the browser.
- **Golden Thread** — trace every release from portfolio epic (Jira Align, SAFe) through to production deployment
- **Release Templates** — save any release as a reusable template; load for next iteration in one click
- **Risk Register** — Likelihood × Impact scoring (1–25), colour-coded severity, feeds into Go/No-Go and PIR
- **Deployment Runbook Builder** — step-by-step with type (Pre-deploy/Deploy/Verify/Rollback), owners, paste-ready output
- **Blackout Calendar** — 6 blackout types including offshore unavailability; UK 2026–27 bank holidays pre-loaded
- **Custom Labels & Badges** — rename delivery models (e.g. Legacy/JSP → Mainframe), add SAP/Salesforce/Oracle; custom severity scales (1–5, S1–S4, Blocker/Critical etc.)
- **6 Practice Scenarios** — fully pre-loaded with realistic scope items, tracking references, risks, runbooks and PIR data

## Integration

Connect from any system that can construct a URL, run JavaScript, or send JSON:

- **12 tracking tools** — Jira, Jira Align, Azure DevOps, ServiceNow, GitHub, Linear, Shortcut, Rally, Monday.com, Confluence, SAFe, Custom
- **Live simulation** — click a scenario (Jira Release Created / ServiceNow CHG Approved / CI/CD Pipeline Complete) to see real-format JSON payloads processed by the tool in real time
- **JS API** — `window.RMTool.loadRelease({...})`, `.exportState()`, `.getBlockers()`, `.isGoReady()`
- **URL parameters** — `?release=Name&type=major&date=2026-09-15&dm=cloud,api,mobile`
- **Paste/Import** — paste any JSON release payload; Import/Export full state as JSON

## Getting started

**Three ways to use it:**

| Option | How | Best for |
|---|---|---|
| **Online** | [Open in browser →](https://Zeus-17.github.io/release-management-intelligence/) | Most users — always the latest version, auto-updates silently |
| **Desktop app (PWA)** | Visit in Chrome/Edge → click ⊞ in address bar → Install | Offline use — works without internet after first install |
| **Local download** | Download HTML + run `python -m http.server 3031` | Corporate / air-gapped environments |

📖 **[Full setup guide](HOW-TO-RUN-LOCALLY.md)**

Once open, the welcome screen gives you four options:
- Take the guided tour (13 steps, ~3 minutes)
- Load a practice scenario (6 realistic, fully pre-loaded)
- Create your first release
- Browse the reference library

**Auto-updates:** When using online or PWA, the service worker automatically clears cached files and loads the latest version every time you open it. No action needed.

```

├── rm-tool-v1.html        ← Main application (single file, no build step)
├── rm-content-v1.json     ← Versioned content layer (ITIL 4, DORA, glossary, regulatory)
├── rm-manifest.json       ← PWA manifest (installable, icons, shortcuts)
├── rm-sw.js               ← Service worker (cache-busting, ensures latest version)
├── rm-showcase.html       ← Product showcase / landing page
├── index.html             ← Redirect to main tool
├── LICENSE                ← Permitted use, regulatory notice, no-liability clause
├── README.md              ← This file
└── HOW-TO.md              ← Maintainer guide (updating content, versioning, GitHub)
```

**The filename `rm-tool-v1.html` never changes.** `v1` = Generation 1. The internal version (`TOOL_VERSION` JS constant) increments with each feature set — you can track this without ever renaming the file.

## Architecture

This tool is deliberately simple:

- **Single HTML file** — the entire application. No npm, no webpack, no Docker.
- **Content JSON** — framework guidance, glossary additions, AI rule overrides and regulatory notices. Updated independently of the app. Fetched automatically on load; changes reach all users without a new deployment.
- **localStorage** — all your data stays in your browser. Export JSON for backup. Import to restore.
- **Service worker** — clears all caches on activate, ensuring every load gets the latest HTML from the network.
